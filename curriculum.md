# Claude 研修計画

## 研修コンセプト

この研修は、Claude の機能を暗記する研修ではありません。
ChatGPT / Gemini などの生成 AI を使ったことがあるが、Claude は初めての受講者を対象に、Claude を使って日々の業務で「考える・調べる・作る・任せる・品質を上げる」方法を学ぶ研修です。

## 研修対象者

- ChatGPT / Gemini などの生成 AI を使ったことがある
- Claude は初めて、またはほとんど使ったことがない
- 業務で Claude を使いたい（コンサルタント・ビジネス職）

## 研修全体の到達目標

- Claude の全体像を理解できる
- Claude の主要機能を業務シーンに紐づけて説明できる
- Chat で日常業務の初速を上げられる
- Research / Office / Artifact / Cowork で成果物を作れる
- Skills / Connectors / Claude Code で品質と再現性を高める考え方を持てる

---

## 研修の全体像

全3回・20セクション — Chat で始め、成果物を作り、品質と再現性を上げる

### 学習の流れ
💬 Chat で始める → 📋 成果物を作る → ✅ 品質と再現性を上げる

### 各回の位置づけ
- **第1回：Claudeで仕事の初速を上げる** — Claude の特徴を理解し、Chat で壁打ち・下書き・要約を業務で使い始める
- **第2回：成果物を作る** — Office・Research・Artifact・Chrome・Cowork で実際の業務成果物を作る力を高める
- **第3回：品質と再現性を上げる** — Skills・MCP・Claude Code で品質を安定させ、チームで再現性高く活用する

---

- プロンプト3要素（前提の共有・タスク・ルール）で的確に指示が出せる
- AI の出力を鵜呑みにせず、確認・修正・責任を持つ姿勢が身についている
- Project と Artifact を使って、業務成果物を作り始められる

### 第2回：成果物を作る（7セクション）
> Claudeを使って文書・表・スライド・調査レポート・触れる成果物まで、業務成果物を自分で作れるようになる

**到達目標：**
- Claude で作れる成果物の種類（文書・表・スライド・調査・チャート・触れる成果物）を把握している
- Claude in Office（Excel / PowerPoint / Word）で既存業務の質とスピードを上げられる
- Research で調査結果を示唆付きで整理できる
- Artifact で診断ツール・ダッシュボード等の「触れる成果物」を作れる
- Chrome 拡張でブラウザ上の作業を効率化できる
- Cowork でファイル作業を委任できる
- Scheduled tasks で定型業務を自動実行できる

| 2-7 | 定型業務を Scheduled tasks 化する | 講義 | ⏰ 定型業務を自動化する |

### 第3回：品質と再現性を上げる（5セクション）
> Claude活用を個人の便利技で終わらせず、チームで再現性高く使うための仕組みを作る

**到達目標：**
- Skills で業務手順を標準化し、チームで再利用できる
- MCP / Connectors で社内ナレッジを Claude に接続し、出力品質を上げられる
- Claude Code で高度な作業（データ処理・ツール構築）を安全に任せられる
- Claude Code を安全かつ快適に使う基本 TIPS（現状調査・変更対象限定・計画提示・確認報告など）を、指示の型として使える
- CLAUDE.md / Hooks / Subagents で品質を固定し、抜け漏れを防げる
- 個人の便利技を CLAUDE.md・テンプレートとしてチームで再現可能にできる
- チームで Claude 活用を展開する際の運用ルール・共有方法を設計できる

| # | セクション | 種別 | グループ |
|---|---|---|---|
| 3-1 | Skills で手順を標準化する（+ Skills化候補の見極め／運用テスト） | 講義 | 🧩 手順を標準化する |
| 3-2 | MCP / Connectors で社内ナレッジにつなぐ | 講義 | 🔗 ナレッジにつなぐ |
| 3-3 | Claude Code で高度な作業を任せる（+ 始める前の準備／うまく動かない時の戻り道／快適TIPS） | 講義 | 💻 高度な作業を任せる |
| 3-4 | CLAUDE.md / Hooks / Subagents で品質を固定する（+ DESIGN.md 等のルールファイル拡張／チーム再現） | 講義 | ✅ 品質を固定する |
| 3-5 | チーム展開する（+ オンボーディング資料／AI情報のチーム定点観測） | 講義 | 👥 チームで広げる |

**TIPS の位置づけ：**
- 本編に組み込むのは「まず試す TIPS」のみ（VS Code / Cursor 拡張版、現状調査、変更対象限定、計画提示、確認報告、ファイルパス連携、基本拡張機能）
- 「慣れたら使う TIPS」は本編で軽く紹介（Web 記事の Markdown 化、Effort 設定）
- 「補助資料・上級 TIPS」は必須ではなく、必要な人が参照する位置づけ（Bypass Permissions、Downloads 自動コピー、xurl、.claude フォルダ操作など）

**確認クイズ：** 11問（80%以上で合格）
**練習問題：** 7問

## AIチューター（実装済み）

### 仕組み
- Claude.ai の **Project** 機能を活用し、研修受講者がいつでも質問できるチューターを実現
- フローティングボタン（`.tutor-fab`）で全画面から常時アクセス可能（研修モード含む）
- クリックすると Claude.ai Project が新しいタブで開く
- ツールチップで「Claude.ai が開きます — 会話は他のメンバーには見えません」と表示

### 設定
- **Project URL**: `app.js` の `AI_TUTOR_URL` 定数に設定
- **Project 共有**: Team 共有（全研修者がアクセス可能、ただし会話履歴は本人のみ閲覧可能）
- **指示文**: `ai-tutor/project-instructions.md` — プロンプト添削、業務応用相談、ハンズオンサポート

### 運用
- カリキュラム更新時は Project のナレッジに最新の curriculum.md を連携
- 指示文の変更は `ai-tutor/project-instructions.md` を編集し、Claude.ai Project に反映

## 配布方法

### 開発環境 → 配布フォルダ
```
開発用（claude-training-app/）     配布用（Claude Skill Up/）
├── index.html            ──→     ├── START.html（リネーム）
├── css/style.css         ──→     ├── css/style.css
├── js/*.js               ──→     ├── js/*.js
├── assets/ehi-logo.png   ──→     └── assets/ehi-logo.png
├── CLAUDE.md             ✗ 除外
├── curriculum.md          ✗ 除外
├── ai-tutor/              ✗ 除外
└── build-dist.*           ✗ 除外
```

### 手順
1. コンテンツを更新
2. `?v=N` をインクリメント
3. `build-dist.ps1` を実行（PowerShell で右クリック→実行）
4. 生成された `Claude Skill Up/` フォルダを Google Drive 共有ドライブにアップロード（上書き）

### 研修者への案内
> 「**START.html** をダブルクリックしてください」
