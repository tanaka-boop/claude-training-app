# Claude 研修計画

## 目的
800（エイトハンドレッド）のマネージャー以上を対象に、Claude の7つの機能（チャット・プロジェクト・アーティファクト・Design・Cowork・Code・Office）を業務で活用できるスキルを身につける実践型研修。ChatGPT/Gemini経験者を前提とし、Claudeだけが持つ「成果物生成」「業務実行」に重点を置く。研修後は研修アプリ内の自習コンテンツ・AIチューターで継続学習し、アドバンスト・応用編でさらにスキルアップする。

## 研修の設計方針
- **実践型**: Claudeを操作して、実際に成果物を作りながら学ぶ
- **段階的**: チャット → Design/Office/Cowork → Claude Code の順に、できることを広げていく
- **成果物ベース**: 提案書（PPTX）、データ分析（XLSX）、プロトタイプ（HTML）を実際に作る
- **TIPS組み込み**: 各セクションにTIPSを配置し、自習モードでまとめ表示も可能

## 研修アプリ技術スタック
- HTML / CSS / JavaScript のみ（依存パッケージなし）
- SPA構成（index.html をダブルクリックでブラウザ起動）
- Google Drive for Desktop で配布・同期

## アプリ構成

### 全体構造
- **ランディングページ**: モード選択（研修モード / 自習モード）
- **研修モード**: スライド型（1セクション=1画面、講師投影用）
- **自習モード**: Webサイト型（ヘッダーナビ、サイドバー目次）
- **AIチューター**: フローティングボタンから Claude.ai Project に接続（全画面常時表示）
- **進捗トラッカー**: localStorage で学習進捗を保持

### カラー & デザイン
- メインカラー: `#127D70`（EHI ティールグリーン）
- モノトーン＋アクセント1色
- コンサルタント向け：構造化・MECE・2×2マトリクス・Before/After変容図

## AIチューター（実装済み）

### 仕組み
- Claude.ai の **Project** 機能を活用し、研修受講者がいつでも質問できるチューターを実現
- フローティングボタン（`.tutor-fab`）で全画面から常時アクセス可能（研修モード含む）
- クリックすると Claude.ai Project が新しいタブで開く
- ツールチップで「Claude.ai が開きます — 会話は他のメンバーには見えません」と表示

### 設定
- **Project URL**: `app.js` の `AI_TUTOR_URL` 定数に設定
- **Project 共有**: Team 共有（全研修者がアクセス可能、ただし会話履歴は本人のみ閲覧可能）
- **指示文**: `ai-tutor/project-instructions.md` — プロンプト添削、業務応用相談、ハンズオンサポート

### 運用
- カリキュラム更新時は Project のナレッジに最新の curriculum.md を連携
- 指示文の変更は `ai-tutor/project-instructions.md` を編集し、Claude.ai Project に反映

研修アプリをAIでよりインタラクティブにするための機能ロードマップ：

### 🔜 次期実装候補

#### 案1：プロンプトビルダー（API不要）
- 4要素（役割・文脈・制約・出力形式）を穴埋め形式で入力
- 完成したプロンプトをワンクリックでコピー → claude.ai に貼り付け
- Good / Bad の比較表示でプロンプト設計スキルを体感で学ぶ
- **技術難易度：低**（HTML/CSS/JSのみ、API不要）

#### 案2：分岐シナリオ学習（API不要）
- 「クライアントからこんな依頼が来ました。どう対応する？」→ 選択肢分岐
- 選択結果に応じてフィードバックとベストプラクティスを表示
- ケーススタディ形式で実践的な判断力を養う
- **技術難易度：低**（静的コンテンツ、JSで分岐制御）

#### 案3：TIPS インタラクティブ検索（API不要）
- 全セクションのTIPSを分類して一覧表示
- タグ・カテゴリでフィルタリング（プロンプト系 / Excel系 / PPT系 / Claude Code系 / 設定系）
- キーワード検索で必要なTIPSにすぐアクセス
- **技術難易度：低**（JS配列のフィルタリング）

#### 案4：AI添削機能（API必要・中期）
- アプリ内でプロンプトを入力 → AIが4要素の観点で採点・添削
- プロキシサーバー経由でClaude APIを呼び出し
- 受講者のAPIキー不要（サーバー側で管理）
- **技術難易度：中**（簡易サーバー構築が必要）

#### 案5：学習進捗に応じた適応型コンテンツ（長期）
- クイズの正答率・学習パターンに基づいて、苦手分野を自動推薦
- 「次はこのセクションを復習しましょう」レコメンデーション
- **技術難易度：中〜高**

## 配布方法

### 開発環境 → 配布フォルダ
```
開発用（claude-training-app/）     配布用（Claude Skill Up/）
├── index.html            ──→     ├── START.html（リネーム）
├── css/style.css         ──→     ├── css/style.css
├── js/*.js               ──→     ├── js/*.js
├── assets/ehi-logo.png   ──→     └── assets/ehi-logo.png
├── CLAUDE.md             ✗ 除外
├── curriculum.md          ✗ 除外
├── ai-tutor/              ✗ 除外
└── build-dist.*           ✗ 除外
```

### 手順
1. コンテンツを更新
2. `?v=N` をインクリメント
3. `build-dist.ps1` を実行（PowerShell で右クリック→実行）
4. 生成された `Claude Skill Up/` フォルダを Google Drive 共有ドライブにアップロード（上書き）

### 研修者への案内
> 「**START.html** をダブルクリックしてください」

## ビジュアルコンポーネント（CSS）
コンテンツのビジュアル化に使用するCSSコンポーネント：

### 基本コンポーネント
- `.visual-flow` — 横型フロー図
- `.visual-grid` / `.visual-grid.cols-2` / `.cols-3` — アイコン付きカードグリッド
- `.visual-comparison` — 左右対比カード
- `.visual-stat` — 数値ハイライトボックス
- `.visual-steps` — 番号付きステップカード（縦型）
- `.visual-highlight-card` — 重要ポイントカード
- `.tip-box`, `.info-box`, `.warning-box`, `.try-box`, `.example-box` — 装飾ボックス

### ストーリー駆動コンポーネント
- `.visual-matrix` — 2×2 ポジショニングマトリクス（軸ラベル付き、`.active`/`.muted` でセル強調）
- `.visual-before-after` — Before/After 変容図（矢印付き左右比較）
- `.visual-equation` — 数式・方程式（項 × 項 = 結果）
- `.visual-big-message` — 大型メッセージカード（アイコン + 太字メッセージ + サブテキスト）
- `.visual-funnel` — ファンネル図（段階的に幅が狭まる + 離脱ポイント表示）
